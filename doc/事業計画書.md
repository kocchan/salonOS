**作成日：** 2026年2月
**作成者：** 個人開発者
**バージョン：** 1.0

## エグゼクティブサマリー

日本全国26万件以上の美容室のうち、88%が個人〜5名規模の小規模経営である。その多くがHOT PEPPER Beautyへの月3〜30万円の掲載費に依存しており、「やめたいが怖くてやめられない」という慢性的な課題を抱えている。

本サービスは、AIコーディング（Claude Code）を活用した個人開発者が構築する**美容室特化型マイクロSaaS**である。予約管理・顧客カルテ・LINEリピーター自動フォローの3機能をシンプルに統合し、月額4,980円（HOT PEPPERの1/10以下）で提供する。

**目標：開始から6ヶ月でMRR10万円、22ヶ月でMRR50万円**

### 1-2. TAM / SAM / SOM 試算

```
TAM（Total Addressable Market）
  全国美容室 263,000店 × 月額3,000円 × 12ヶ月
  = 約94.7億円/年

SAM（Serviceable Addressable Market）
  個人〜5名規模 231,000店 のうち、デジタルツールに月額支出のある店舗（推計40%）
  = 92,400店 × 月額3,000円 × 12ヶ月
  = 約33.3億円/年

SOM（Serviceable Obtainable Market）
  ローンチ後3年で2%シェア獲得を目標とした場合
  = 1,848店 × 月額4,980円 × 12ヶ月
  = 約1.1億円/年（ARR）
```

### 1-6. MVP機能定義

#### Must（MVP・最初のリリースに必須）

| # | 機能 | 理由 |
|---|------|------|
| 1 | **ネット予約受付（LINE連携・Webフォーム）** | 電話予約の機会損失が最大のペイン |
| 2 | **予約台帳（カレンダービュー）** | 転記ミス防止・視認性の確保 |
| 3 | **顧客カルテ（基本情報＋施術履歴）** | スタッフ間引き継ぎ・薬剤ミス防止 |
| 4 | **カラー・薬剤記録欄** | 美容室特化の最大の差別化ポイント |
| 5 | **来店から3ヶ月経過アラート** | 休眠顧客フォローの自動化 |
| 6 | **LINEセミカスタマイズ送信** | 顧客名・前回施術を差し込んだ擬似個別送信 |
| 7 | **スマートフォン対応UI（PWA）** | 施術の合間にスマホで操作する実態に対応 |

#### やってはいけない過剰機能

| 機能 | 理由 |
|------|------|
| AI似合わせ診断 | 美容師のプロとしての判断を機械に委ねることへの強い拒否感 |
| EC機能（物販） | スコープクリープ。今解くべき課題ではない |
| SNS自動投稿 | 美容師は「自分のセンスで投稿したい」という強いこだわりがある |
| 労務管理・シフト自動作成 | 別カテゴリのプロダクト。何をするアプリか伝わらなくなる |

> **MVP判断基準：「この機能がないと、お客様が解約するか？」Noであれば含めない。**

### 2-3. 競合分析

#### ポジショニングマップ

```
                  高機能
                    |
    カロッテ        |    Beauty Merit
    Mindbody       |   （美容特化・高機能）
                    |
高価格 ─────────────+─────────────── 低価格
                    |
  HOT PEPPER Beauty |   【本サービス】
  （掲載型・別軸）  |  STORES予約
                    |  Airリザーブ・Square
                  低機能
```

#### 差別化の3軸

1. **vs. 汎用ツール（Airリザーブ・STORES予約・Square）**
   - 美容室特有のカラー薬剤記録・施術履歴管理・再来店サイクルフォローに特化

2. **vs. 高機能特化ツール（カロッテ・Beauty Merit・Mindbody）**
   - 「1人で30分以内にセットアップできる」シンプルさ。月3万円以下の価格

3. **vs. HOT PEPPER Beauty**
   - 顧客データをオーナー自身が所有できる。利害が一致している（依存させない）

#### 持続的競争優位（Moat）

1. **顧客データの蓄積**：使い続けるほどサロン固有のカルテが蓄積し、乗り換えコストが上昇
2. **LINEエコシステムとの深い統合**：後発が追いにくい連携品質
3. **美容師コミュニティの口コミネットワーク**：横のつながりが強く、成功事例が伝播しやすい

### 3-2. ブランドポジショニング

#### コンセプトステートメント
> **「HOT PEPPERからの卒業を、ひとりの個人開発者が全力で支える。」**

#### 個人開発者の強みをどう伝えるか

「大企業にはできない3つのこと」として発信する。

1. **開発者本人がサポートする**
   - 「問い合わせたら、作った人間が直接答える」体験は法人SaaSでは絶対に再現できない
   - 発信文例：「バグ報告してくれたら、翌日には直します。それが個人開発の速さです。」

2. **ユーザーの声が翌週の機能になる**
   - 大企業は機能リリースまで数ヶ月かかる。個人開発は1週間で反映できる

3. **売り込まない。一緒に考える。**
   - 「このツールが合わなければ、HOT PEPPERに戻った方がいい場合もある」と正直に言える

### やってはいけないこと（まとめ）

| 落とし穴 | 理由 |
|---------|------|
| 「全業種対応」でスタートする | 誰にも刺さらないツールになる |
| プロダクトを完璧にしてからマーケする | 出遅れ。最低限動く状態で10社に使ってもらう |
| 月980円などの極端な低価格設定 | 解約率が上がる逆効果。3,000円以上が継続の心理的閾値 |
| 広告費をPhase 1に使う | CAC/LTVの比率が見えない段階での広告は赤字製造機 |
| SNSで「宣伝」しか発信しない | 投稿の8割は業界の有益情報。残り2割でツールに言及 |
| 解約申し出に値引きで対応する | 「値引き要求すれば安くなる」と学習させてしまう |
| 月10万達成後すぐに別業種展開する | 同一業種でMRR30万達成後に横展開するのが原則 |
